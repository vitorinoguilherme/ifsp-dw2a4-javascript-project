Javascript Boas e Más Práticas de Desenvolvimento

Sempre ativar "use strict"

Os navegadores são muito complacentes com a sintaxe JavaScript. Com a opção "use strict" ativada garantimos que erros silenciosos sejam obtidos

Use "===" Ao Invés de "=="

Ao comparar se dois operandos tem o mesmo tipo e valor, o === retorna verdadeiro e !== retorna falso

See the Pen Comparação by Guilherme Vitorino (@vitorinoguilherme) on CodePen.

Evite Variáveis Globais

A razão é que todo arquivo JavaScript incluso na página roda no mesmo escopo.
Caso tenha variáveis globais ou funções no seu código e decida inserir scripts contendo os mesmos nomes das variáveis e funções eles vão sobrescrever o código escrito anteriormente

See the Pen Globais by Guilherme Vitorino (@vitorinoguilherme) on CodePen.
